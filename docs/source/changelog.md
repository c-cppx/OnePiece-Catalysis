# Changelog

`onepiece-studio` keeps its release notes in the repository root file
`CHANGELOG.md`.

This file is the authoritative release history for the package and should be
updated whenever a new version is prepared for publication.

## Current Release

The current documented release is:

- `1.0.0` — marks the first stable release with cleaned release engineering,
  stricter package boundaries, security/lint hooks, and verified backend/UI
  build artifacts.

## What Belongs In The Changelog

For this package, release notes should focus on:

- scientific behavior changes
- workflow or project-model changes
- UI behavior changes that affect analysis reproducibility
- package/distribution changes
- documentation and release-process updates

The root changelog is preferred over scattered ad hoc release notes so that:

- PyPI releases have one clear history
- scientific behavior changes stay visible
- small research teams can audit package evolution quickly
